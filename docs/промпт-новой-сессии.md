# Промпт новой сессии

Составь стартовый промпт для новой Codex/Claude-сессии по текущей задаче Terminal UI на iOS.

Перед составлением промпта обязательно:

- прочитай `AGENTS.md` / `CLAUDE.md`;
- вызови `docs_search` по теме Terminal iOS navigation jank / scroll reset / UIKit container;
- прочитай `/Users/fedor/Desktop/vs-code/habit-tracker/docs/terminal-navigation-jank-investigation-2026-06-15.md`;
- проверь текущий `git status`, активную ветку и последние изменения в `TerminalControlUI.swift` / `TerminalControlStore.swift`.

В промпте для новой сессии явно зафиксируй:

- цель: довести Terminal navigation до стабильной архитектуры `projects -> tabs -> chat` без jank, без сброса scroll и с нормальным интерактивным back-свайпом;
- не повторять старые подходы: bitmap/snapshot overlay как основной путь, SwiftUI `ScrollView` + кастомная page-анимация, глобальный SwiftUI snapshot всего Terminal store в `UIViewControllerRepresentable`;
- UIKit-контейнер должен владеть горизонтальной навигацией и вертикальными scroll views;
- SwiftUI root не должен читать весь `tabsByProject`, `statusByTab`, `runningTabs` ради передачи в UIKit;
- нельзя делать `reloadData()` / `invalidateLayout()` на каждый `updateUIViewController`, prefetch, status или SSE publish;
- обновления UIKit-списков делать по узким diff/key: только если изменился состав строк или видимые поля конкретных строк;
- scroll offset должен жить в UIKit scroll view, anchors в store только как fallback/restore, а не как команда на каждый переход;
- во время push/back не монтировать тяжёлые live SwiftUI-деревья и не включать row-level heavy controls;
- chat detail можно временно оставить SwiftUI-hosted, но проекты и вкладки должны быть UIKit-owned;
- после изменений обязательно запускать `./deploy.sh`.

Попроси новую сессию сначала коротко описать план реализации и риски, затем сделать изменения небольшими проверяемыми шагами. Если она предлагает переписать всё сразу или передать весь store snapshot в SwiftUI wrapper, это неправильный план.
