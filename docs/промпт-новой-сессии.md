# Промпт новой сессии

Составь стартовый промпт для новой Codex/Claude-сессии, если следующая задача снова касается
Terminal UI на iOS.

Перед составлением промпта обязательно:

- прочитай `AGENTS.md` / `CLAUDE.md`;
- вызови `docs_search` по теме Terminal iOS navigation jank / scroll reset / UIKit container;
- прочитай `docs/knowledge/fact-voice-chat-tab.md::Terminal mode внутри AI Chat` и
  `::Terminal navigation: UIKit container, стабильные shells, logs`;
- прочитай `docs/knowledge/fix-ios-stability.md::Terminal navigation jank`;
- прочитай `docs/methodology/переносимый-дизайн.md::Нативный компонент vs кастом`;
- проверь текущий `git status`, активную ветку и последние изменения в
  `TerminalControlUI.swift` / `TerminalControlStore.swift` / `VoiceChatUI.swift`.

В промпте для новой сессии явно зафиксируй:

- текущий baseline: Terminal navigation уже живёт в `TerminalUIKitNavigationController`;
  `projects/tabs` — `UITableView`, `chat` — SwiftUI-hosted detail;
- не повторять старые подходы: bitmap/snapshot overlay как основной путь, SwiftUI `ScrollView` + кастомная page-анимация, глобальный SwiftUI snapshot всего Terminal store в `UIViewControllerRepresentable`;
- UIKit-контейнер должен и дальше владеть горизонтальной навигацией и вертикальными scroll views;
- SwiftUI root не должен читать весь `tabsByProject`, `statusByTab`, `runningTabs` ради передачи в UIKit;
- нельзя делать `reloadData()` / `invalidateLayout()` на каждый `updateUIViewController`, prefetch, status или SSE publish;
- обновления UIKit-списков делать по узким diff/key: только если изменился состав строк или видимые поля конкретных строк;
- scroll offset должен жить в UIKit scroll view, anchors в store только как fallback/restore, а не как команда на каждый переход;
- во время push/back не монтировать тяжёлые live SwiftUI-деревья;
- skeleton cells допустимы только для холодной загрузки без данных, а не как маска перехода;
- если пользователь присылает новые логи, сначала сравнить `label="uikit ..."` / `dropped` /
  `worst` / `[term-swipe-geo]`, а не начинать рефактор по ощущениям;
- после изменений обязательно запускать `./deploy.sh`.

Попроси новую сессию сначала коротко описать план реализации и риски, затем сделать изменения
небольшими проверяемыми шагами. Если она предлагает переписать всё сразу, вернуть SwiftUI
snapshot/bitmap path или передать весь store snapshot в SwiftUI wrapper, это неправильный план.
